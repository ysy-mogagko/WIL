## 1. 디자인 분석 및 초기 평가 (Desktop - Sidebar.png)

### 👍 주요 장점 (Strengths)
* **포근하고 일관된 컬러 톤앤매너**: 짙은 카키 그린과 부드러운 크림 베이지 배경색을 매치하여 장시간 달력을 보아도 눈이 피로하지 않는 내추럴하고 감성적인 무드를 제공합니다.
* **사이드바를 활용한 올인원 기능**: 메인 달력 외에도 우측 영역에 미니 달력, ToDo, Memo, Diary, D-Day 섹션을 배치하여 단순 스케줄러를 넘어선 종합적인 '개인 플래너'로서의 정체성을 잘 드러냈습니다.
* **깔끔한 일정 표현**: 일정의 성격(발표과제, 조별과제, 저녁약속 등)에 따라 다른 컬러 칩을 사용하고, 텍스트가 넘칠 때 `show +3`과 같은 생략 표현을 사용해 그리드의 일관성을 유지했습니다.

### 🧐 초기 UI/UX 개선 포인트
1. **날짜 데이터 표기 오류**: 디자인 에셋 복사 과정에서 더미 데이터(`1` 등)가 그대로 남거나 주차별 흐름이 맞지 않는 현상이 발견되어 실제 날짜 데이터로의 수정이 필요했습니다.
2. **미니 달력의 정보 중복**: 메인 뷰가 '월별보기'일 때, 우측 사이드바에서도 동일한 월의 미니 달력이 노출되어 화면 공간의 비효율성이 존재했습니다.
3. **주말 및 공휴일 시각적 구분 부족**: 모든 요일과 날짜가 검은색 계열로 되어 있어 요일 직관성이 낮았습니다. (토요일/일요일에 블루/레드 계열의 내추럴 톤 적용 제안)
4. **날짜 숫자 위치 변경 필요**: 셀 중앙에 숫자가 배치되어 있어 일정 텍스트가 많아질 경우 겹치거나 가독성이 떨어질 우려가 있어 좌측 상단으로 배치를 제안했습니다.
5. **뷰 전환 아이콘 직관성**: '월별보기' 옆의 햄버거 메뉴 아이콘(≡)을 드롭다운 화살표 메뉴(`▼` 또는 `∨`)로 변경하여 기능 인지도를 높이도록 제안했습니다.

## 2. 주별보기(Weekly View) & 일별보기(Daily View) 확장 설계

기존 톤앤매너를 유지하며 사용성을 확장한 주별 및 일별 보기의 설계 방향입니다.

### 📅 주별보기 (Weekly View)
* **구조**: 7열 세로 그리드 레이아웃 시스템을 도입하여 일주일의 일정 밀도를 한눈에 파악할 수 있도록 설계했습니다.
* **주말 가독성**: 평일과 다르게 일요일은 **Soft Red**, 토요일은 **Soft Blue** 계열의 톤다운된 컬러를 적용해 요일 경계를 명확히 했습니다.
* **오늘 하이라이트**: 사용자가 오늘 일정을 즉시 인지할 수 있도록 당일 날짜 상단이나 배경에 부드러운 그린 포인트 인디케이터를 추가했습니다.

### 🕐 일별보기 (Daily View)
* **구조**: 좌측에 24시간 타임라인 룰러(Timeline Ruler)를 배치하여 시간 단위의 조밀한 스케줄 관리가 가능하도록 설계했습니다.
* **포커스 패널**: 우측 사이드바 영역에 당일의 핵심 목표(Focus) 및 연관 메모 레이아웃을 강조하여 하루의 생산성을 극대화할 수 있도록 지원합니다.

## 3. 월별 달력(Monthly View) 최종 개선안

사용자의 피드백을 수용하여 최종적으로 도출된 월별 달력의 수정 및 개선 핵심 내용입니다.

1. **실제 날짜 정렬**: 2023년 5월 달력 기준(5월 1일 월요일 시작)에 맞추어 실제 일자 데이터를 완벽하게 배치했습니다.
2. **숫자 위치 재배치 (중앙 ➔ 좌측 상단)**: 숫자를 셀의 왼쪽 위 구석으로 이동시켜, 일정이 아래 방향으로 깔끔하고 여유롭게 쌓일 수 있는 가독 공간을 확보했습니다.
3. **사이드바 위젯 최적화**: 중복되던 미니 달력을 과감히 제거하고, 디데이 트래커(**Upcoming D-Day 위젯**)로 대체하여 사이드바의 실용성을 높였습니다.
4. **여백 및 컴포넌트 튜닝**: 
   * 좌측 ISO 주차 레이블(25, 26, 27...)과 달력 그리드 사이에 충분한 패딩(여백)을 주어 답답함을 해소했습니다.
   * 상단 내비게이션의 뷰 체인저 버튼을 '월별보기 ▼' 형태로 변경하여 드롭다운 인터랙션의 직관성을 완성했습니다.

## 2. 타겟 유저

- 국내 ETF로 장기 분산 투자 중인 개인 투자자
- KODEX, ACE, RISE, TIGER 등 국내 상장 ETF 보유자
- 리밸런싱을 해야 한다는 건 알지만 언제 해야 할지 모르는 사람

## 3. 핵심 가치 제안

전일 종가 기준으로 매일 아침 비중을 자동 계산
  → 목표 비중에서 ±N% 벗어나면 텔레그램 알림 발송
  → "지금 뭘 얼마나 사고팔면 되는지" 자동 계산

> 장기 ETF 투자자에게 실시간 시세는 필요 없음.
> "이번 주에 리밸런싱 해야 하나?" 라는 질문에 매일 아침 답해주는 것이 핵심.

## 4. 기술 스택

### 백엔드
- **Spring Boot** — REST API 서버
- **MySQL** — 사용자 포트폴리오, 설정 저장
- **Redis** — 시세 캐싱 (당일 API 응답 캐시)
- **Spring Scheduler** — 매일 아침 비중 계산 및 알림 스케줄링

### 프론트엔드
- **Next.js** — 대시보드 + 설정 페이지 (심플하게 유지)

### 외부 연동
- **공공데이터포털 API** — 금융위원회 주식시세 / 증권상품시세(ETF) 정보
- **텔레그램 Bot API** — 알림 발송

### 인프라
- **AWS EC2** — 서버
- **AWS RDS** — MySQL
- **AWS ElastiCache** — Redis

## 5. MVP 기능 (4주)

> 이 6가지가 완성되면 실제로 쓸 수 있는 서비스가 됨.

### 🔵 P0-1. 회원가입 / 로그인

- 이메일 + 비밀번호 회원가입
- 소셜 로그인 (카카오 / 구글)
- JWT 인증 + Refresh Token

### 🔵 P0-2. 포트폴리오 등록

- ETF 종목 검색 (종목명 / 종목코드)
- 보유 수량 + 평균 매수가 입력
- 목표 비중 % 입력 (합계 100% 유효성 검증)
- 종목 추가 / 수정 / 삭제

### 🔵 P0-3. 비중 현황 대시보드

- 현재 비중 vs 목표 비중 나란히 비교
- 종목별 괴리율 표시 (예: +3.2%, -5.1%)
- 전일 종가 기준 평가금액 + 총 자산
- 공공데이터포털 API 연동 (매일 오후 1시 이후 갱신)

### 🔵 P0-4. 리밸런싱 계산기

- "지금 어떻게 사고팔면 목표 비중이 되는가" 자동 계산
- 추가 납입 금액 입력 시 "이달에 뭘 얼마나 살지" 추천
- 매수 / 매도 수량 + 금액 안내

### 🔵 P0-5. 텔레그램 알림

- 텔레그램 봇 연동 (봇 토큰 발급 + Chat ID 연결)
- 괴리율이 임계값 초과 시 자동 알림 발송
- 매일 아침 9시 정기 포트폴리오 리포트 발송
- 알림 ON / OFF 설정

### 🔵 P0-6. 알림 설정

- 괴리율 임계값 설정 (기본값: ±5%)
- 알림 수신 시간 설정
- 알림 일시정지 기능

## 6. 2단계 기능 (5~8주)

> 유저 반응 보고 우선순위 조정.

### 🟢 P1-1. 수익률 트래킹

- 종목별 손익 계산 (현재가 - 평균 매수가 × 수량)
- 전체 포트폴리오 수익률 (%)
- 매수가 대비 등락 표시

### 🟢 P1-2. 리밸런싱 히스토리

- 알림 이력 저장 및 조회
- 알림 받은 후 실제 리밸런싱 실행 여부 체크
- "그때 했더라면 얼마나 달랐을까?" 시뮬레이션

### 🟢 P1-3. 다중 포트폴리오 관리

- ISA / CMA / IRP 계좌 각각 독립 관리
- 계좌별 목표 비중 분리 설정
- **유료 플랜 전환점** — 무료는 포트폴리오 1개 제한

### 🟢 P1-4. 구글 애드센스 연동

- 무료 플랜 페이지에 광고 노출
- 유료 플랜 가입 시 광고 제거
- 공공데이터포털 기반이라 합법적 수익화 가능

## 9. 데이터 소스

### 공공데이터포털 금융위원회 API

| API | 제공 데이터 | 갱신 주기 | 용도 |
|---|---|---|---|
| 주식시세정보 | 주식 전일 종가, 등락률, 거래량 | 영업일 익일 오후 1시 | 일반 주식 종목 시세 |
| 증권상품시세정보 | ETF 시가, 종가, 거래량, 변동률 | 영업일 익일 오후 1시 | ETF 종목 시세 |
| KRX상장종목정보 | 종목코드, 종목명, 시장구분 | 일 1회 | 종목 검색 |

> **주의**: 모든 데이터는 실시간이 아닌 전일 종가 기준.
> 장기 ETF 투자자 대상이므로 서비스 특성상 전일 종가로 충분함.

### 시세 캐싱 전략

매일 오후 2시 → 공공데이터포털 API 호출
  → Redis에 당일 시세 저장 (TTL: 익일 오후 2시까지)
  → 유저 요청 시 Redis에서 조회 (API 직접 호출 최소화)
